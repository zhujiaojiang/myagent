# 常用英文术语对照(Git / 开发)

> 编程绕不开英文,但都是**高频重复**的"常客"。认熟这几十个,就不云里雾里了。
> 看到不认识的英文词,先来这里翻。

## 一、Git 命令(动作)

| 英文 | 中文 | 干嘛 |
|---|---|---|
| git | 吉特(工具名) | 版本控制工具 |
| status | 状态 | 看现在什么情况 |
| add | 添加 | 把改动放进暂存区 |
| commit | 提交 | 拍一个存档点 |
| push | 推送 | 把存档传到云端 |
| pull | 拉取 | 从云端取下最新 |
| clone | 克隆 | 把整个仓库拷到本地 |
| diff | 差异 | 看改了哪几行 |
| log | 日志 | 看历史存档 |
| restore | 还原 / 撤回 | 丢掉还没提交的改动 |
| branch | 分支 | 代码的平行版本线 |
| merge | 合并 | 把分支并到一起 |

## 二、状态词(常出现在 git status 里)

| 英文 | 中文 |
|---|---|
| modified | 已修改 |
| staged | 已暂存(放进暂存区了) |
| untracked | 未跟踪(git 还没开始管它) |
| changes not staged for commit | 改动还没暂存 |
| up to date | 已是最新(和云端同步) |
| nothing to commit, working tree clean | 没东西可提交,干净 |

## 三、名词

| 英文 | 中文 | 说明 |
|---|---|---|
| repository / repo | 仓库 | 存代码 + 历史的地方 |
| remote | 远程 | 云端那份仓库 |
| origin | 远程的默认昵称 | 通常就指你的 GitHub 仓库 |
| main / master | 主分支(名字) | 默认主线 |
| commit(名词) | 一个存档点 | 每个有唯一编号(哈希 hash) |
| credential | 凭证 | 登录用的"通行证" |
| authorize | 授权 | 同意某程序访问你的账号 |

## 四、平台 / 生态

| 英文 | 是什么 |
|---|---|
| GitHub | 托管代码的网站 |
| HuggingFace | 托管 AI 模型的网站 |
| ModelScope(魔搭) | 国内的模型网站 |
| PyPI | Python 包的仓库(pip install 从这下载) |
| Public / Private | 公开 / 私有 |

## 五、报错 / 提示词

| 英文 | 意思 |
|---|---|
| warning | 警告(通常无害,提醒一下) |
| error | 错误(这个要处理) |
| fatal | 致命错误(更严重,必须处理) |
| failed | 失败了 |
| not found | 没找到 |
| permission denied | 没权限 |
